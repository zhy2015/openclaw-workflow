# Feishu Card Usage Baseline

## When to prefer interactive cards
- Review conclusions with clear verdicts and next steps
- Task/status updates with concise structured sections
- Todo confirmations / execution summaries
- Alerts or high-signal notices

## When to keep plain text
- Casual back-and-forth chat
- One-line confirmations
- Low-value short acknowledgements

## Default card style
- Header: short title, color by intent
  - blue: normal update
  - green: done / success
  - orange: warning / needs attention
  - red: failure / blocker
- Body: one compact summary block + optional next-step block
- Footer note: lightweight source / state tag

## Preferred templates
1. Review verdict card
   - Title
   - Verdict
   - What changed
   - Remaining concerns
2. Status update card
   - Current state
   - Completed
   - Next
3. Alert card
   - Problem
   - Impact
   - Suggested action
