# STARTUP_SKILLS_CARD

- 技能策略：默认不展开全量技能说明
- 路由原则：先按 task node / 具体任务筛选，再按需读取单个 SKILL.md
- 启动期仅保留：渠道规则、当前任务最相关技能、必要工具契约
- 延迟加载：治理文档、历史技能、长说明书、项目专用技能
- 控制面归属：跨 skill 的 contract / routing / visibility / lifecycle 归 `skill-governance`
- 目标：把技能从“全量暴露”改为“检索式暴露”
- Feishu 会话补充：当前已验证可发送 interactive card；在结构化高价值回复中可优先考虑卡片，而非只发纯文本
- Review 工作流补充：收到外部 review 时优先套用 `receiving-code-review`；当仓库已基础可用后，再按需切 `architecture-designer`
